## 2024-10-31 - [Path Traversal in Proxy Server]
**Vulnerability:** Found a critical path traversal vulnerability in `api/proxy.cjs` where the static assets endpoint `/assets/` used user input (`req.url`) directly in `path.join` without normalizing and validating that the resolved path stayed within the intended directory. This allowed unauthorized reading of sensitive files like `.env` using payloads like `/assets/../../.env`.
**Learning:** The proxy server's naive implementation of path joining failed to consider encoded characters (`%2e%2e`) and URL structure differences (like query strings). Express or typical web servers handle this natively, but raw Node.js `http` modules require explicit validation.
**Prevention:** Always normalize the resolved file path using `path.normalize()` and explicitly check that the resolved path `startsWith` the absolute base path (`distDir + path.sep`) before serving the file. Also, ensure the URL is decoded and stripped of query parameters (`split('?')[0]`).
