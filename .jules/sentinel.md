## 2026-07-08 - Fix Path Traversal in Static File Serving
**Vulnerability:** The proxy server's `/assets/` static file handler used `path.join(__dirname, '..', 'dist', req.url)` without validating that the final resolved path remained within the intended `dist` directory, allowing attackers to use `../` in the URL to read arbitrary files on the system (e.g., `/etc/passwd`).
**Learning:** Using `path.join` with unsanitized user input (`req.url`) directly exposes the filesystem. The `path.join` method resolves `../` segments, effectively allowing directory traversal. Even if `req.url` seems constrained by a routing prefix, the underlying system still processes the raw path segments.
**Prevention:** Always use `path.resolve` combined with `path.normalize` to get the absolute path, ensure the user input is treated as relative (by prepending `.`), decode the URL components, and explicitly validate that the final path starts with the intended base directory including a path separator (e.g., `filePath.startsWith(basePath + path.sep)`).
