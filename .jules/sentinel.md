## 2026-07-04 - Fix path traversal in static asset server
**Vulnerability:** The static asset serving route in `api/proxy.cjs` (`/assets/`) was vulnerable to path traversal. By simply appending `../` sequences, arbitrary files on the filesystem could be read (e.g., `curl --path-as-is http://localhost:3002/assets/../../api/proxy.cjs` successfully returned the proxy source code).
**Learning:** Naive concatenation of user-provided paths (e.g., using `path.join(__dirname, '..', 'dist', req.url)`) without verifying if the fully resolved, absolute path actually remains within the intended root directory is a critical vulnerability. Attackers can use this to access source files, credentials, or system files.
**Prevention:** Always use a combination of URL decoding (`decodeURIComponent`), path normalization (`path.normalize`), and absolute path resolution (`path.resolve`). Crucially, perform an explicit boundary check using `startsWith(basePath + path.sep)` to ensure the calculated path does not escape the designated base directory.
