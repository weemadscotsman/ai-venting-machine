## 2024-06-30 - Fix Path Traversal in Static Asset Server
**Vulnerability:** The proxy server in `api/proxy.cjs` was serving static assets by blindly joining the request URL (`req.url`) to the base directory (`dist`). An attacker could supply directory traversal sequences (like `/assets/../../../etc/passwd`) via HTTP requests to read arbitrary files from the host server.
**Learning:** Node.js `path.join` natively resolves `../` segments without constraining them to the base path. Blindly trusting `req.url` to serve files without validating the resulting absolute path's origin causes severe arbitrary file read vulnerabilities.
**Prevention:** Always validate that the final resolved path resides within the intended base directory. This can be achieved by computing the absolute path using `path.join` (and optionally `path.normalize` / `path.resolve`), decoding the URI components to handle URL-encoded sequences, and asserting `resolvedPath.startsWith(basePath + path.sep)`. Reject requests (e.g., HTTP 403 Forbidden) if the check fails.
