## 2026-07-11 - Path Traversal Vulnerability in Static Asset Serving
**Vulnerability:** A path traversal vulnerability existed in the static file serving logic in `api/proxy.cjs` because `path.join(__dirname, '..', 'dist', req.url)` was used directly. This allowed an attacker to read arbitrary files by requesting paths like `/assets/../../api/proxy.cjs`.
**Learning:** `path.join` does not sanitize user input against path traversal. Using it to map user-provided URLs to disk can allow an attacker to escape the intended base directory using `../` sequences, especially when the server does not decode the URL first or check if the resolved file is actually inside the intended directory.
**Prevention:** To prevent path traversal, first `decodeURI()` the user path. Then safely calculate the absolute `basePath` and the absolute target `filePath` via `path.normalize(path.resolve(basePath, '.' + decodedUrl))`. Finally, explicitly validate that the target file path starts with the base path, e.g., `filePath.startsWith(basePath + path.sep)`.
