## 2024-07-11 - Unnecessary React Re-renders and I/O due to new array references in Interval State Setters
**Learning:** In a React application that synchronizes state with `localStorage` on every state change (e.g., via `useEffect(() => { localStorage.setItem(...) }, [state])`), unconditionally returning a new array reference from a state setter within an interval causes both a React re-render and expensive synchronous disk I/O, even if the actual data hasn't conceptually changed.
**Action:** When updating arrays or objects inside intervals, iterate to check if any underlying values actually require modification. Use a `hasChanges` flag and only return the new reference if true; otherwise, return the previous reference to short-circuit the re-render and side-effect cascades.
