## 2024-05-24 - Unconditional Array Mappings in Intervals Trigger Unnecessary Renders and I/O
**Learning:** Using `.map()` inside a state setter (like `setAgents`) during a frequent interval creates a new array reference unconditionally. In React, this causes unnecessary re-renders. Furthermore, if a `useEffect` hook persists the state to localStorage (e.g., `useEffect(() => { localStorage.setItem('vm_agents_cannon_chaos_v2', JSON.stringify(agents)); }, [agents]);`), this new array reference will trigger disk I/O every tick, even if no values actually changed, leading to compounding performance degradation over time.
**Action:** When updating arrays in intervals, always iterate with a `hasChanges` flag. If no internal values change, return the `prev` array reference to bail out of the React update cycle and prevent cascading side-effects like disk I/O.
