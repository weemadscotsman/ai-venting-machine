## 2024-07-22 - Prevented unbounded re-rendering in list components
**Learning:** In React applications that render lists of volatile items incrementally (like chat logs rendering via `setInterval`), appending a new item or updating the state of a single item causes all previously rendered components in the list to re-render if they are rendered inline inside the mapping function. Also, passing non-memoized functions or objects down to child components creates new references each render, further breaking React's ability to bail out.
**Action:** Extract inline list items into their own components wrapped in `React.memo()`. Utilize `useCallback` for functions passed to list items, combined with the "latest ref" pattern (`useRef` + `useEffect` to cache latest state/props) to maintain a stable function reference without capturing stale closures.
